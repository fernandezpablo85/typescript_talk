Frontend Errors

Stack Traces
  • Ocurre en el device del usuario
  • Se puede shipear con window.onError (pero hay ciertos issues con los CDN)
  • Por lo general minificada. Requiere sourcemaps.
  • El modelo asincrono y de callbacks dificulta la lectura (hooks, promesas que se resuelven, código de framework)

Backend Errors

Logs (& Other telemetry tools)
  • They Exist. On your servers.
  • 3rd party libraries logs in most languages (java, python, others)
  • Log severity (debug, info, warn, error) configurable at runtime
  • syslog, dmesg, tcpdump, etc. on the running machine
  • online diagnostic tools: perf, strace, jvisualvm

Frontend Errors

Logs (& Other telemetry tools)
  • They (usually) don't exist. If they do, they're on the user console
  • Need to ship them back for analysis (window.onError, 3rd party tools)
  • You may also ship additional relevant info (user-agent, screen resolution, etc)
